# 风险趋势功能 - 导入错误修复和启动指南

## 问题诊断

### 原始错误
```
ImportError: attempted relative import with no known parent package
```

### 错误原因
多个文件使用了相对导入（`.utils`），但直接运行脚本时无法识别父包

## 已修复的文件

### 1. analyzer.py
**修复前**：
```python

from .utils.risk_calculation_service import get_risk_calculator
```

## 常见问题

### Q1: 端口 5000 被占用
**错误**：`Address already in use`

**解决**：
```bash
# 查找占用端口的进程
netstat -ano | findstr :5000

# 终止进程
taskkill /PID <进程 ID> /F
```

### Q2: 模块导入错误
**错误**：`ModuleNotFoundError`

**解决**：
```bash
# 确保在正确的目录
cd C:\Users\User\Documents\xx_IndexAnalyse

# 或者添加路径
set PYTHONPATH=%PYTHONPATH%;C:\Users\User\Documents\xx_IndexAnalyse\web\backend
```

### Q3: 数据文件缺失
**错误**：`File not found`

**解决**：
确保数据文件存在于正确的位置

## 技术细节

### 导入修复原则
1. **包内导入**：使用绝对导入（`utils.module`）
2. **包外导入**：使用相对导入或添加路径
3. **动态导入**：在函数内部导入时使用绝对路径
