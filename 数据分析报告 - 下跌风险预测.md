# 下跌风险预测数据分析报告

## 问题 1：数据不匹配原因分析

### 观察到的现象
- **下跌风险预测模块**显示：
  - 次日下跌概率：**51%**
  - 5 日内下跌概率：**49%**

- **综合风险评估模块**显示：
  - 次日下跌概率：**42%**
  - 5 日内下跌概率：**58%**

### 根本原因

**两个模块使用了完全不同的数据源：**

| 模块 | 数据源 | 位置 | 计算方式 |
|------|--------|------|----------|
| 下跌风险预测 | **前端模拟数据** | `index.html` 第 1248-1249 行 | `T+1 = seededRandom(dateSeed) * 60 + 20`<br>`T+5 = seededRandom(dateSeed + 1000) * 50 + 30` |
| 综合风险评估 | **后端真实计算** | `/api/predict` 接口 | 25 因子 Logit 回归模型 |

### 代码证据

**前端模拟数据生成**（`index.html` 第 1234-1259 行）：
```javascript
// 生成最近 100 个交易日的预测数据（使用固定种子确保数据一致性）
const generatePredictionData = () => {
  const data = [];
  const today = new Date();
  
  for (let i = 99; i >= 0; i--) {
    const date = new Date(today);
    date.setDate(date.getDate() - i);
    const dateStr = date.toISOString().split('T')[0];
    
    // 使用日期作为种子，确保同一日期生成的数据始终一致
    const dateSeed = parseInt(dateStr.replace(/-/g, ''));
    
    // 生成模拟数据（使用固定种子）
    const t1Prob = seededRandom(dateSeed) * 60 + 20; // 20-80%
    const t5Prob = seededRandom(dateSeed + 1000) * 50 + 30; // 30-80%
    
    data.push({
      date: dateStr,
      t1Prob: parseFloat(t1Prob.toFixed(2)),
      t5Prob: parseFloat(t5Prob.toFixed(2))
    });
  }
  
  return data;
};
```

**后端真实计算**（`risk_calculation_service.py` 第 319-409 行）：
```python
def predict_downside_probability(self, factors, df=None):
    # 计算综合风险分
    base_risk, adjusted_risk, trend_adj = self.calculate_comprehensive_risk(factors, df)
    
    # 计算波动率
    volatility_20d = df['change_pct'].rolling(20).std().iloc[-1]
    
    # 计算动量
    momentum_1d = df['change_pct'].iloc[-1]
    momentum_5d = df['change_pct'].tail(5).sum()
    
    # T+1 下跌概率 (Logit 模型)
    linear_combination_t1 = (
        alpha_0 + alpha_1 * adjusted_risk + 
        alpha_2 * (adjusted_risk ** 2) + 
        alpha_3 * volatility_20d + 
        alpha_4 * momentum_1d
    )
    prob_t1 = 1 / (1 + np.exp(-linear_combination_t1))
    
    # T+5 下跌概率 (扩展 Logit 模型)
    linear_combination_t5 = (
        beta_0 + beta_1 * adjusted_risk + 
        beta_2 * (adjusted_risk ** 2) + 
        beta_3 * volatility_20d + 
        beta_4 * momentum_5d + 
        beta_5 * macro_uncertainty + 
        beta_6 * (adjusted_risk * momentum_5d)
    )
    prob_t5 = 1 / (1 + np.exp(-linear_combination_t5))
    
    return {
        'prob_t1': prob_t1,
        'prob_t5': prob_t5,

    }
```

## 问题 2:100 个交易日下跌概率趋势 - 数据与公式匹配性

### 当前实现

**前端趋势图数据**（`index.html` 第 1248-1249 行）：
```javascript
const t1Prob = seededRandom(dateSeed) * 60 + 20; // 范围 20-80%
const t5Prob = seededRandom(dateSeed + 1000) * 50 + 30; // 范围 30-80%
```

**后端计算公式**（`risk_calculation_service.py` 第 347-394 行）：

**T+1 模型**：
```
ln(P/(1-P)) = α₀ + α₁·R_adjusted + α₂·R_adjusted² + α₃·Volatility + α₄·Momentum

参数：
- α₀ = -1.5 (截距项)
- α₁ = 2.0  (风险分线性项系数)
- α₂ = 0.5  (风险分二次项系数)
- α₃ = 0.03 (波动率系数)
- α₄ = -0.02 (动量系数)
```

**T+5 模型**：
```
ln(P/(1-P)) = β₀ + β₁·R_adjusted + β₂·R_adjusted² + β₃·Volatility + β₄·Momentum_5d 
            + β₅·MacroUncertainty + β₆·(R_adjusted · Momentum_5d)

参数：
- β₀ = -1.2
- β₁ = 1.8
- β₂ = 0.4
- β₃ = 0.04
- β₄ = -0.03
- β₅ = 0.05
- β₆ = -0.02
```

### 匹配性结论

**❌ 完全不匹配**

1. **前端模拟数据**：
   - 使用固定范围的随机数生成
   - 范围：T+1 (20-80%), T+5 (30-80%)
   - 与真实市场数据、25 因子模型无任何关系

2. **后端真实计算**：
   - 基于 25 个核心因子的 Logit 回归
   - 考虑：综合风险分、波动率、动量、宏观不确定性
   - 与真实市场数据强相关

3. **实际观察到的 25%-75% 波动范围**：
   - 在理论范围 (20-80%) 内
   - 属于正常的随机波动
   - **但不代表真实市场预测**

## 总结与建议

### 核心问题

1. ✅ **数据不匹配原因已定位**：
   - 下跌风险预测：前端模拟数据
   - 综合风险评估：后端真实计算
   - 两者完全独立，无任何关联

2. ✅ **100 日趋势图数据与公式不匹配**：
   - 前端：随机数生成（20-80% 范围）
   - 后端：25 因子 Logit 模型
   - 实际观察到的 25%-75% 波动在正常范围内

3. ✅ **逻辑合理性说明**：
   - 当前数据为模拟数据，无真实市场含义
   - 仅用于 UI 占位和可视化展示
   - 需在文档中明确标注

### 修改建议（按优先级）

#### 高优先级 - 统一数据源
- **问题**：两个模块数据不一致，用户体验差
- **方案**：两个模块都调用后端 `/api/predict` 接口
- **影响**：解决数据不一致问题

#### 中优先级 - 实现历史回溯
- **问题**：100 日趋势图为模拟数据
- **方案**：后端实现历史数据回溯计算
- **影响**：提升数据真实性和可信度

#### 低优先级 - 添加说明文档
- **问题**：用户可能误解模拟数据为真实预测
- **方案**：添加明确说明文字
- **影响**：降低用户误解风险
