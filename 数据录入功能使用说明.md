# 数据录入功能使用说明

## 📋 功能概述

数据录入功能支持两种模式：
1. **单条录入** - 手动输入单条行情数据
2. **批量导入** - 粘贴 JSON 数组或东方财富 JSONP 格式数据

## 🎯 保存方式

### 方式一：后端服务自动保存（推荐）

**前提条件：后端服务必须运行**

#### 启动后端服务的方法：

**方法 1：双击批处理文件（最简单）**
```
双击：启动后端服务.bat
```

**方法 2：命令行启动**
```bash
cd web/backend
python start_backend.py
```

**验证后端是否运行：**
- 浏览器访问：http://localhost:5000/api/health
- 看到响应表示后端已启动

**保存流程：**
1. 点击"📝 数据录入"按钮
2. 填写或粘贴数据
3. 点击"💾 保存"
4. 系统自动保存到 `web/frontend/index_history_data.json`
5. 页面立即刷新显示新数据

---

### 方式二：手动下载文件（后端未运行时）

**如果后端服务未运行，系统会自动下载 JSON 文件**

**保存流程：**
1. 点击"📝 数据录入"按钮
2. 填写或粘贴数据
3. 点击"💾 保存"
4. 浏览器自动下载 `index_history_data.json` 文件
5. 手动将下载的文件复制到 `web/frontend/` 目录覆盖原文件
6. 刷新页面生效

---

## 📝 数据格式

### 单条录入格式

填写以下字段：
- **日期**（必填）：选择日期
- **收盘价**（必填）：例如 2047.89
- **涨跌幅**（必填）：例如 0.21
- **成交额**（可选）：单位亿元，例如 1.40
- **换手率**（可选）：单位%，例如 11.32

---

### 批量导入格式

#### 格式 1：标准 JSON 数组

```json
[
  {"date": "2024-01-03", "close": 1458.32, "change_pct": 0.85, "amount": 0.18, "turnover": 3.21},
  {"date": "2024-01-04", "close": 1462.15, "change_pct": 0.26, "amount": 0.19, "turnover": 3.35}
]
```

#### 格式 2：东方财富 JSONP 格式

直接复制东方财富的响应数据，例如：
```javascript
jQuery35108745723541970376_1772087141861({"data":{"klines":["2024-02-07,1000.90,907.53,1000.90,894.44,41970859,22549057482.00,10.65,-9.25,-92.47,4.87"]}})
```

系统会自动解析并转换为标准格式。

## 🔍 常见问题

### Q1: 保存时提示"文件保存失败"

**原因：后端服务未运行**

**解决方案：**
1. 启动后端服务（双击 `启动后端服务.bat`）
2. 重新保存数据
3. 或者使用自动下载的文件，手动覆盖

### Q2: 如何验证数据已保存？

**方法：**
1. 打开 `web/frontend/index_history_data.json` 文件
2. 检查最后几条数据是否是刚保存的
3. 刷新浏览器页面，查看是否显示新数据

### Q3: 数据保存后页面没有更新？

**解决方案：**
1. 强制刷新浏览器（Ctrl + F5）
2. 检查浏览器控制台是否有错误
3. 确认 JSON 文件格式正确

---

## 💡 最佳实践

1. **启动后端服务再保存** - 避免手动复制文件的麻烦
2. **定期备份数据** - 复制 `index_history_data.json` 到安全位置
3. **批量导入前验证** - 使用 JSON 格式验证工具检查数据
4. **保留原始数据** - 保存东方财富原始响应，便于重新解析

---

## 🛠️ 技术细节

### 数据流向

```
数据录入 → 验证格式 → 更新页面 → 保存文件
                          ↓
                    后端运行？
                          ↓
              是 → API 保存 → 成功提示
                          ↓
              否 → 下载文件 → 手动覆盖
```

### 文件位置

- **静态数据文件**：`web/frontend/index_history_data.json`
- **风险趋势数据**：`web/frontend/risk_history_data.json`
- **后端服务**：`web/backend/simple_app.py`

### API 端点

- **保存数据**：`POST http://localhost:5000/api/save_index_data`
- **健康检查**：`GET http://localhost:5000/api/health`
