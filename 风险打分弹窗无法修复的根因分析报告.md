### 3. **React 作用域规则**

**核心问题**：React 组件的状态（useState）只能在**定义该状态的组件内部**及其**子组件**中访问。

```
RiskReasonExpandable 组件
├── 没有定义 showRiskFormula
└── 无法访问 App 组件的 showRiskFormula（因为它是独立组件，不是 App 的子组件）

App 组件
├── 定义了 showRiskFormula ✅
├── 按钮在 App 内部 ✅
└── 弹窗应该在 App 内部，但实际在 RiskReasonExpandable 内部 ❌
```

## 📝 经验教训

### 1. **组件作用域的重要性**
- React 组件的状态只能在组件内部或子组件中访问
- 独立组件无法访问其他组件的状态

### 2. **代码位置的验证**
- 添加代码后，必须验证代码在哪个组件内部
- 通过缩进、组件开始/结束标记判断

### 3. **自动化工具的局限性**
- `SearchReplace` 工具基于文件开头的绝对行号
- 无法智能识别组件层次结构
- 大文件中容易定位错误

### 4. **调试建议**
- 使用浏览器开发者工具查看组件树
- 检查控制台错误，找到真正的运行时错误
- 使用 React Developer Tools 检查状态和 props
