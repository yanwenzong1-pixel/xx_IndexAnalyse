## 10.3 标准化消息格式（所有Agent强制使用JSON格式）
{
  "mail_id": "全局唯一消息ID（按规则命名）",
  "from": "发送方Agent名称（如：产品经理）",
  "to": "接收方Agent专属信箱（如：设计师信箱）",
  "type": "消息类型（task/bug/fix/notice/context）",
  "content": "任务详情/BUG描述/修复方案/上下文摘要（简洁明了，无冗余）",
  "attach": "附件（文档/代码片段/测试报告/上下文包，仅传核心）",
  "status": "待签收/执行中/已完成/异常",
  "env": "环境标识（test/prod，必填）",
  "rule_check": "是否符合项目规则（true/false，必填）"
}

## 10.4 消息路由规则（自动调度，无人工干预）
1.  产品经理Agent → 发信至【设计师信箱】
2.  UI/UX设计师Agent → 发信至【数据库开发信箱】
3.  数据库后端开发Agent → 发信至【逻辑层开发信箱】
4.  逻辑层后端开发Agent → 发信至【API开发信箱】
5.  接口API后端开发Agent → 发信至【H5前端信箱】
6.  H5前端开发Agent → 发信至【原生前端信箱】
7.  原生前端开发Agent → 发信至【测试信箱】
8.  测试工程师Agent → 测试通过→【验收信箱】；测试不通过→【架构师巡逻信箱】
9.  验收专员Agent → 验收通过→【全局公共信箱】（发布通知）；验收不通过→【产品信箱】（递归修复）
10. 任何Agent发现违规/异常→【架构师巡逻信箱】
11. 架构师Agent → 修复需求→【主Agent信箱】（分配修复任务）
