# 第三章 敏捷开发5S规则（融入全流程）
1.  Sort（整理）：需求、代码、文档、上下文按类别归档，清理冗余文件、无效代码、过期上下文，保持项目整洁。
2.  Set in Order（整顿）：严格遵循单向开发流程，规范文件存放、上下文分级，确保开发有序、可追溯。
3.  Shine（清洁）：代码无硬编码、无上下文污染、无全局变量滥用、无代码异味，上下文无冗余、无无效信息。
4.  Standardize（标准化）：统一命名规范、API调用规范、上下文格式、Mailbox消息格式，实现全流程标准化。
5.  Sustain（维持）：架构师Agent定期巡逻，持续维护规则落地、上下文规范，确保开发流程长期合规。

# 第四章 项目文件结构规范（全栈通用，严格对标）
## 4.1 根目录（仅允许存在以下内容，严禁新增无关文件）
项目根目录
-src/ # 核心业务代码唯一存放目录（所有开发代码必须放入）
-config/ # 全局配置目录（常量、环境变量、密钥，严禁硬编码）
-tests/ # 测试用例目录，与 src 目录结构完全同级对应，同步更新
-docs/ # 文档目录（接口文档、需求文档、设计文档、上下文管理文档）
-logs/ # 日志文件目录（自动生成，不上传 Git，严禁存放其他内容）
-.env # 环境变量配置文件（密钥、端口等，不上传 Git）
-README.md # 项目说明文档（含规则、流程、上下文管理说明）
-依赖配置文件 # 如 package.json、requirements.txt，仅保留核心依赖
