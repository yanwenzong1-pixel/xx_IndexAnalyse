# 第八章 环境隔离与发布规则（安全可控）
1.  环境强隔离：测试（test）环境与生产（prod）环境实现物理隔离，严禁互通、严禁污染，数据、配置、上下文分开管理。
2.  测试优先原则：所有开发、自查、巡逻修复、测试操作，仅在测试环境执行，严禁在生产环境直接操作。
3.  一次性发布规范：测试环境全量测试通过后，方可一次性发布覆盖生产环境，严禁分批发布、违规发布。
4.  生产环境保护：架构层严禁任何开发操作直接影响生产环境，仅允许验收通过后的发布操作，确保生产环境稳定。

# 第九章 Vibe Coding 上下文压缩与管理规范（企业级标准）
## 核心原则（必守，降低Token成本，提升效率）
1.  必要性原则：仅保留AI生成、开发执行必需的核心信息，删除所有冗余、无效、无关内容。
2.  最小可用原则：上下文按需提供，不整包上传、不冗余堆砌，实现Token消耗降低60%~90%。
3.  结构化原则：上下文统一格式、分级管理，确保AI可识别、可复用、可追溯。
4.  可管控原则：上下文可复现、可更新、可清理，权限分级管控，避免混乱。

## 上下文压缩规则（强制执行）
1.  文件结构压缩：严禁上传node_modules、dist、logs等冗余目录/文件；仅保留核心链路文件（前端page→component→api，后端controller→service→model）；路径采用极简格式（如[Page] dashboard/user-management）。
2.  代码内容压缩：删除代码中的注释、空行、无关函数、冗余逻辑；函数/组件仅保留名称、入参、出参、功能摘要及关键逻辑；类型/接口仅保留关键字段；样式仅保留类名层级，无需完整样式代码。
3.  依赖与提示词压缩：package.json等依赖文件仅保留核心依赖，删除冗余依赖；提示词采用固定模板（项目信息→技术栈→需求→上下文→核心逻辑），简洁明了。
4.  自动化处理：通过脚本自动过滤黑名单文件、删除冗余内容、生成上下文摘要、标准化打包，提升效率。
