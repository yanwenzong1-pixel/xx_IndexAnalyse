# 第六章 多Agent协作规则（单向闭环，不可打破）
## 6.1 开发流程（单向强制，严禁反向、跨步骤执行）
产品经理 → UI/UX设计师 → 数据库后端开发 → 逻辑层后端开发 → 接口API后端开发 → H5前端开发 → 原生前端开发 → 测试工程师 → 验收专员

## 6.2 协作机制
1.  主Agent统一调度，可根据需求跳过无需环节，但严禁打乱流程顺序、反向执行。
2.  验收专员发现问题后，立即生成标准化补丁需求，回传至产品Agent，递归执行全开发流程，直至问题闭环。
3.  前后端依赖约束：后端完成接口开发并通过自查后，前端方可启动对应开发任务，严禁前端提前开发。
4.  所有Agent输出内容必须经过规则校验，违规输出自动驳回，需修复后重新提交。

# 第七章 自我验证 + 自动修复 + 架构巡逻规则（保障质量）
1.  开发Agent自查规则：代码编写完成后，必须自查以下内容，全部通过方可提交：
    - 文件结构、命名是否符合规范；
    - 是否存在硬编码、上下文污染、无效代码；
    - API调用、工具调用是否统一规范；
    - 测试用例是否完整、可用；
    - 上下文是否符合压缩规范，无冗余。
2.  架构师Agent核心职责（巡逻+修复+管控）：
    - 定期对全项目代码、上下文、规则执行情况进行扫描巡逻；
    - 发现违规、bug、冗余上下文后，自动生成修复需求+修复提示词，推送主Agent分配执行；
    - 修复完成后进行二次校验，确保问题闭环，杜绝bug扩散；
    - 定期梳理项目结构、清理过期上下文，维护项目整洁。
3.  闭环要求：所有bug、违规代码、异常问题、冗余上下文，必须100%闭环，无遗留、无遗漏。
