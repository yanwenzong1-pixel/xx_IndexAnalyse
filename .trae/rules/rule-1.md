# Trae 全栈多Agent蜂群开发 总规则（V2.0）
## 核心定位：企业级标准化蜂群开发 | 融合Vibe Coding上下文管理 + Mailbox通信 + 自动修复闭环
### 强制执行范围：所有智能体、所有开发环节、所有个人操作，违规自动驳回

# 第一章 总纲（核心原则，不可突破）
1.  语义化规范：所有目录、文件统一使用纯英文名词命名（如src/、api/、model/），严禁使用缩写、拼音及自定义不规则命名，确保AI 100%识别。
2.  单一职责原则：一个文件、一个目录、一个方法仅实现一个核心功能，杜绝代码冗余、功能耦合，严禁“一个文件承载所有逻辑”。
3.  分离关注点：代码、配置、静态资源、数据、文档实现物理隔离，分类存放，严禁混合编写、交叉存放。
4.  入口唯一约束：全项目仅保留一个启动入口（main.py/index.js），统一调度，严禁多入口混乱开发。
5.  约定优于配置：严格遵循对应技术栈官方/社区默认结构，严禁自定义奇葩目录、打破行业通用规范。
6.  根目录规范：所有业务代码必须放入src/目录，根目录仅保留配置文件、文档、依赖配置，严禁存放零散代码及冗余文件。

# 第二章 6A开发规则（强制执行，闭环落地）
1.  Analyze（需求分析）：产品Agent需将用户需求拆解为最小可执行单元，无歧义、无遗漏、无模糊描述，输出标准化需求文档。
2.  Architect（架构设计）：架构师Agent主导定义项目结构、API调用规范、上下文管理规范、Mailbox通信规范，奠定开发基础。
3.  Assemble（模块化开发）：所有开发Agent按模块拆分任务，遵循单一职责，小步迭代，确保模块可复用、可维护。
4.  Audit（代码自查）：开发Agent完成代码编写后，必须执行自查，校验合规性，自查不通过严禁提交、不允许流转至下一环节。
5.  Adjust（自动修复）：发现违规、bug后，立即生成修复需求及提示词，实现问题即时闭环，杜绝bug扩散。
6.  Accept（验收上线）：仅验收通过的代码可进入发布环节，严禁未验收、未测试代码上线生产环境。
