## 1. 需求对齐（1A）

### 1.1 需求背景

当前"下跌风险预测"模块的近 100 个交易日下跌概率趋势图使用前端模拟数据（`seededRandom`），与"综合风险评估"模块使用的后端真实计算（25 因子 Logit 模型）数据不一致，导致：

1. 两个模块显示的"次日下跌概率"、"5 日内下跌概率"数值不匹配
2. 趋势图数据与计算公式完全脱节
3. 用户体验差，数据可信度低

### 1.2 需求目标

**核心目标**：实现历史数据回溯功能，使前端趋势图展示真实的模型计算结果

**具体目标**：
1. 后端支持回溯计算历史 100 个交易日的下跌概率
2. 前端调用真实 API 替代模拟数据
3. 两个模块数据源统一，数值一致
4. 保持页面加载性能不受影响

### 1.3 功能需求

#### FR-001：历史数据回溯接口
- **描述**：后端提供 `/api/predict/history` 接口

### 2.4 核心算法设计

#### 2.4.1 历史回溯计算逻辑

```python
def calculate_history(days=100):
    """
    计算历史 N 日的下跌概率
    
    Args:
        days: 回溯天数
        
    Returns:
        list: 每日预测结果
    """
    # 1. 获取历史数据（需要 days + 252 天，因为因子计算需要 252 天窗口）
    market_data = fetch_market_data(days=days + 252)
    
    results = []
    
    # 2. 从后往前循环（最近的日期在前）
    for i in range(days):
        # 当前日期索引
        current_idx = len(market_data) - 1 - i
        
        # 3. 截取到当前日期的历史数据（避免未来函数）
        df_up_to_current = market_data[:current_idx + 1]
        
        # 4. 计算 25 因子数据
        factors = factor_service.get_all_factors(df_up_to_current)
        
        # 5. 计算下跌概率
        result = risk_calculator.predict_downside_probability(
            factors, 
            df_up_to_current
        )
        
        # 6. 添加日期信息
        result['date'] = df_up_to_current['date'].iloc[-1].strftime('%Y-%m-%d')
        
        results.append(result)
    
    # 7. 反转结果（最近的日期在前）
    results.reverse()
    
    return results
```

#### 2.4.2 因子计算注意事项

**关键问题**：因子计算需要 252 天历史窗口

**解决方案**：
- 回溯 100 日数据 → 需要获取 352 日市场数据
- 第 1 日的因子计算使用前 252 日数据
- 确保不使用未来数据（look-ahead bias）
