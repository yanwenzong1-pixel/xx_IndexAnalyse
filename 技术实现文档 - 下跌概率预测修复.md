# 下跌概率预测数据一致性修复 - 技术实现文档

## 问题描述

下跌概率趋势图（次日 T+1 和 5 日 T+5）每次页面刷新都会显示完全不同的数据，但按照业务逻辑，基于固定的历史数据和公式，预测结果应该是稳定的。

## 问题根因

在 `RiskPrediction` 组件中，使用了 `Math.random()` 生成随机数据：

```javascript
const t1Prob = Math.random() * 60 + 20; // 20-80%
const t5Prob = Math.random() * 50 + 30; // 30-80%
```

这导致每次页面刷新时都会生成全新的随机数据，无法保持一致性。

## 解决方案

### 1. 使用固定种子生成可重复的随机数

实现了 `seededRandom` 函数，基于种子值生成确定性的"随机"数：

```javascript
const seededRandom = (seed) => {
  const x = Math.sin(seed++) * 10000;
  return x - Math.floor(x);
};
```

**原理**：
- 对于相同的种子值，始终返回相同的结果
- 使用日期字符串作为种子（如：20250309）
- 确保同一日期的预测数据始终一致

### 2. 基于日期生成种子

对于每个交易日，使用其日期字符串作为种子：

```javascript
const dateSeed = parseInt(dateStr.replace(/-/g, ''));
const t1Prob = seededRandom(dateSeed) * 60 + 20;
const t5Prob = seededRandom(dateSeed + 1000) * 50 + 30;
```

**优势**：
- 同一日期的数据永远相同
- 不同日期的数据有差异（通过偏移种子值）
- 数据看起来仍然具有"随机性"

### 3. 添加 localStorage 缓存机制

实现了 24 小时缓存策略：

```javascript
// 检查缓存
const cachedData = localStorage.getItem('predictionData');
const cachedTimestamp = localStorage.getItem('predictionDataTimestamp');

if (cachedData && cachedTimestamp) {
  const timestamp = parseInt(cachedTimestamp);
  const now = Date.now();
  const hoursPassed = (now - timestamp) / (1000 * 60 * 60);
  
  if (hoursPassed < 24) {
    setPredictionData(JSON.parse(cachedData));
    return;
  }
}

// 更新缓存
localStorage.setItem('predictionData', JSON.stringify(newData));
localStorage.setItem('predictionDataTimestamp', Date.now().toString());
```

**优势**：
- 减少重复计算
- 提高页面加载速度
- 确保 24 小时内数据完全一致
- 即使清除浏览器缓存，24 小时内也能保持一致
