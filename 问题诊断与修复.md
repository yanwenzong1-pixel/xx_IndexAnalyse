# 风险趋势功能问题诊断与修复

## 问题描述
页面刷新后 Tab 标签已生成，但内容无法加载。

## 已实施的修复

### 1. 添加调试日志
在 `index.html` 的 TabChart 组件中添加了详细的调试日志：
- API 请求开始日志
- API 响应状态日志
- API 返回结果日志
- 数据成功/失败日志
- 错误详细日志

### 2. 添加缓存加载逻辑
添加了页面加载时自动检查 localStorage 缓存的功能：
```javascript
React.useEffect(() => {
  const cachedData = localStorage.getItem('riskHistoryData');
  const cachedTimestamp = localStorage.getItem('riskHistoryTimestamp');
  
  if (cachedData && cachedTimestamp) {
    const timestamp = parseInt(cachedTimestamp);
    const now = Date.now();
    const hoursPassed = (now - timestamp) / (1000 * 60 * 60);
    
    // 缓存 24 小时
    if (hoursPassed < 24) {
      console.log('[RiskTab] 使用缓存数据');
      setRiskData(JSON.parse(cachedData));
      return;
    }
  }
}, []);
```

### 3. 创建测试页面
创建了独立的测试页面 `test-risk-data.html` 用于：
- 验证后端 API 是否正常工作
- 检查返回数据格式
- 测试 localStorage 缓存

## 常见问题及解决方案

### 问题 1：后端服务未启动
**症状**：Network 标签显示请求失败（ERR_CONNECTION_REFUSED）

**解决方案**：
```bash
python web/backend/simple_app.py
```

### 问题 2：CORS 跨域错误
**症状**：Console 显示 CORS policy 错误

**解决方案**：在 `simple_app.py` 中启用 CORS（当前已注释，如需启用请取消注释）

### 问题 3：数据格式不匹配
**症状**：图表不显示，Console 显示数据错误

**检查数据格式**：
```javascript
// 期望的数据格式
{
  "success": true,
  "data": [
    {
      "date": "2024-01-01",
      "risk_score": 6.5,  // 必须是数字
      "risk_level": "high"  // 必须是字符串
    }
  ]
}
```

### 问题 4：缓存过期
**症状**：每次都要重新加载数据

**解决方案**：缓存有效期为 24 小时，如需刷新数据可清除缓存：
```javascript
localStorage.removeItem('riskHistoryData');
localStorage.removeItem('riskHistoryTimestamp');
location.reload();
```

### 问题 5：图表组件未加载
**症状**：显示"加载中..."但数据已获取

**检查**：
1. RiskTrendChart.js 是否正确引入
2. 组件是否导出：`window.RiskTrendChart = RiskTrendChart;`
3. 数据是否为数组且长度>0
