## 一、现状分析

### 1.1 当前问题

**现象**：
- **最新下跌概率**（综合风险评估模块）：42%（后端真实计算）
- **近 100 日趋势图最新值**（下跌风险预测模块）：~80%（前端模拟数据）
- **两者差异**：最大可达 40 个百分点，用户体验差，数据不一致

### 1.2 根本原因

| 模块 | 数据源 | 计算方式 | 状态 |
|------|--------|----------|------|
| **综合风险评估** | 后端 `/api/predict` | 25 因子 Logit 模型 | ✅ 真实数据 |
| **下跌风险预测** | 前端模拟（降级） | `seededRandom() * 60 + 20` | ⚠️ 模拟数据 |

**核心问题**：
1. 前端尝试调用 `/api/predict/history?days=100` 失败后，降级到模拟数据
2. 最新下跌概率硬编码在 HTML 中（第 1532 行：`51%`）
3. 趋势图使用模拟数据，但与最新值无关联

### 1.3 代码位置

**综合风险评估模块**（真实数据）：
- 前端显示：`index.html` 第 3462-3478 行（硬编码 42% / 58%）

## 二、对齐目标

### 2.1 核心目标

**统一算法**：两个模块都使用相同的后端 25 因子 Logit 模型计算

**数据一致**：
- 最新下跌概率 = 历史趋势图最新一日数值
- 误差范围：0%（完全一致）

**用户体验**：
- 消除数据不一致带来的困惑
- 提升数据可信度
- 保持 UI 响应速度

### 2.2 具体指标

| 指标 | 当前状态 | 目标状态 |
|------|----------|----------|
| **最新 T+1 概率** | 硬编码 51% | 后端实时计算（动态） |
| **趋势图最新值** | 模拟 ~80% | 后端历史回溯（动态） |
| **数据一致性** | ❌ 不一致 | ✅ 完全一致 |
| **API 调用** | 降级到模拟 | ✅ 成功调用 |
| **加载时间** | 即时 | < 2 秒（可接受） |

## 三、技术方案

### 3.1 方案概述

**核心思路**：统一使用后端真实数据，前端动态渲染

**实施策略**：
1. 确保后端历史回溯 API 正常工作
2. 前端统一调用后端 API，移除模拟数据
3. 最新下跌概率从历史数据中提取（最新一日）
4. 添加加载状态和错误处理

```javascript
// 计算预期跌幅（基于后端逻辑）
const calculateExpectedDecline = (prob_t1, volatility_20d) => {
  // 分段线性模型
  let expected_decline;
  if (prob_t1 < 0.3) {
    expected_decline = -0.3;
  } else if (prob_t1 < 0.5) {
    expected_decline = -0.6;
  } else if (prob_t1 < 0.7) {
    expected_decline = -1.2;
  } else {
    expected_decline = -2.0;
  }
  
  // 根据波动率调整
  const adjustment_factor = Math.max(0.5, Math.min(2.0, volatility_20d / 2.0));
  expected_decline *= adjustment_factor;
  
  return expected_decline.toFixed(1);
};

// 计算置信区间
const calculateDeclineRange = (prob_t1, volatility_20d) => {
  let lower, upper;
  if (prob_t1 < 0.3) {
    lower = -0.5;
    upper = -0.1;
  } else if (prob_t1 < 0.5) {
    lower = -0.9;
    upper = -0.3;
  } else if (prob_t1 < 0.7) {
    lower = -1.7;
    upper = -0.7;
  } else {
    lower = -2.8;
    upper = -1.2;
  }
  
  const adjustment_factor = Math.max(0.5, Math.min(2.0, volatility_20d / 2.0));
  lower *= adjustment_factor;
  upper *= adjustment_factor;
  
  return `${lower.toFixed(1)}% ~ ${upper.toFixed(1)}%`;
};
```
