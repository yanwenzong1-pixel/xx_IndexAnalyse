# 风险趋势功能增强 - 实现说明

## 📋 功能概述

本次更新实现了两个核心功能，优化了风险趋势页面的用户体验和系统稳定性：

1. **静态缓存数据** - 初次启动时不依赖后端服务
2. **后台服务状态监控** - 实时显示后端服务运行状态

---

## ✨ 功能一：静态风险历史数据

### 实现方式

**数据文件位置**：`web/frontend/risk_history_data.json`

**数据特征**：
- 包含 252 个交易日的风险评分历史数据
- 时间范围：2025-03-17 至 2026-03-13
- 每条记录包含：
  - `date`: 日期（ISO 8601 格式）
  - `risk_score`: 风险分数（1-10 分）
  - `risk_level`: 风险等级（low/medium/high/very_high）

**数据分布**：
- 低风险 (low): 约 40%
- 中等风险 (medium): 约 35%
- 高风险 (high): 约 20%
- 极高风险 (very_high): 约 5%

### 加载逻辑

```javascript
// 优先级顺序
1. 静态 JSON 文件（./risk_history_data.json）
   ↓ 如果不存在或加载失败
2. 后端 API（http://localhost:5000/api/risk/history?days=60）
```

### 代码位置

**RiskTab 组件** (`index.html:1908-1965`):
```javascript
const fetchRiskData = React.useCallback(async (showLoading = false) => {
  // 优先尝试加载静态 JSON 文件数据
  console.log('[RiskTab] 尝试加载静态数据文件...');
  try {
    const staticResponse = await fetch('./risk_history_data.json');
    if (staticResponse.ok) {
      const staticData = await staticResponse.json();
      console.log('[RiskTab] 静态数据加载成功，条数:', staticData.length);
      setRiskData(staticData);
      setRiskLoading(false);
      return;
    }
  } catch (staticErr) {
    console.log('[RiskTab] 静态文件不存在或加载失败，尝试从 API 获取...');
  }
  
  // 如果静态文件不存在，尝试从 API 获取
  console.log('[RiskTab] 开始请求 API 数据...');
  // ... API 请求逻辑
}, []);
```

### 优势

✅ **无需等待后端启动** - 页面加载即可显示数据  
✅ **降低后端压力** - 减少不必要的 API 调用  
✅ **提升用户体验** - 秒级加载，无等待时间  
✅ **离线可用** - 即使后端服务不可用，也能查看历史数据

## ✨ 功能二：后台服务状态监控

### 实现的功能

1. **服务状态检测**
   - 实时检测后端服务是否运行
   - 每 30 秒自动检查一次状态
   - 状态显示：运行中 / 未连接

2. **服务日志展示**
   - 记录服务连接事件
   - 显示时间戳和日志类型
   - 保持最近 20 条日志

3. **智能提示**
   - 后端未运行时显示启动指南
   - 加载中的友好提示
   - 加载失败的错误提示

### UI 展示

**位置**：风险趋势页面底部

**显示内容**：

### 代码实现

**状态定义** (`index.html:3085-3086`):
```javascript
const [backendServiceStatus, setBackendServiceStatus] = React.useState(false);

```

**日志函数** (`index.html:3409-3415`):
```javascript
const addServiceLog = (message, type = 'info') => {
  const now = new Date();
  const timeStr = `${now.getHours().toString().padStart(2, '0')}:${now.getMinutes().toString().padStart(2, '0')}:${now.getSeconds().toString().padStart(2, '0')}`;
  setServiceLogs(prev => [...prev.slice(-19), { time: timeStr, message, type }]);
};
```

**状态检测** (`index.html:3417-3461`):
```javascript
React.useEffect(() => {
  const checkBackendStatus = async () => {
    try {
      const response = await fetch('http://localhost:5000/api/health', { 
        method: 'GET',
        headers: { 'Accept': 'application/json' }
      });
      if (response.ok) {
        const data = await response.json();
        setBackendServiceStatus(true);
        addServiceLog('后端服务连接成功', 'success');
        return true;
      } else {
        throw new Error('服务响应异常');
      }
    } catch (error) {
      setBackendServiceStatus(false);
      addServiceLog('后端服务未连接', 'error');
      return false;
    }
  };

  // 首次检查
  checkBackendStatus().then(isRunning => {
    if (!isRunning) {
      addServiceLog('正在等待后端服务启动...', 'warning');
    }
    loadData();
  });

  // 每 30 秒检查一次服务状态
  const statusInterval = setInterval(() => {
    checkBackendStatus();
  }, 30000);

  // ...
}, []);
```

### 日志类型

- **success** (绿色): 后端服务连接成功
- **error** (红色): 后端服务未连接
- **warning** (黄色): 正在等待后端服务启动
- **info** (灰色): 普通信息

## 🎯 使用场景

### 场景一：首次启动系统

1. 用户打开页面
2. 立即显示静态风险历史数据（秒级加载）
3. 后台服务状态显示"未连接"
4. 用户看到启动指南
5. 后端服务启动后，状态自动更新为"运行中"

### 场景二：后端服务不可用

1. 用户打开页面
2. 静态数据正常显示
3. 服务状态显示"未连接"
4. 用户可以查看历史数据，但无法获取最新数据
5. 服务日志显示连接失败记录

### 场景三：后端服务正常运行

1. 用户打开页面
2. 静态数据立即显示
3. 服务状态检测通过，显示"运行中"
4. 可以选择从 API 获取最新数据
5. 服务日志显示连接成功记录
