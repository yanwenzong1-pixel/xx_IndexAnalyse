## 1. 影响范围分析

### 1.1 代码变更总结

根据开发 Agent 的修复方案，本次修复主要针对以下两个核心问题：

#### 1.1.1 价格指数图表刷新修复
**涉及文件**：
- `web/frontend/src/components/PriceChart.jsx` - 核心修复
- `web/frontend/src/App.jsx` - 强制刷新支持

**主要变更**：
1. 优化 useEffect 依赖管理，使用 useCallback 包装函数
2. 改进图表实例生命周期管理，增加 DOM 可见性检查
3. 添加数据深度比较逻辑（deepCompareArrays）
4. 实现强制刷新机制（forceRefreshKey）
5. 使用 requestAnimationFrame 确保 DOM 渲染完成

#### 1.1.2 下跌概率预测数据一致性修复
**涉及文件**：
- `web/frontend/index.html` - RiskPrediction 组件

**主要变更**：
1. 实现 seededRandom 函数，基于日期生成确定性随机数
2. 添加 localStorage 缓存机制（24 小时有效期）
3. 确保同一日期的预测数据始终一致

### 1.2 影响模块分析

### 1.3 调用链分析

```
用户操作/F5 刷新
    ↓
App.handleRefresh / loadData
    ↓
setRefreshKey(prev => prev + 1)
    ↓
PriceChart.forceRefreshKey 变化
    ↓
forceRefresh() 执行
    ↓
销毁旧实例 → 创建新实例 → 渲染图表
```

### 1.4 数据流分析

```
数据加载流程：
1. App.jsx 加载数据（mockData / API）
2. 数据验证（结构完整性检查）
3. 保存到 localStorage
4. 传递给 PriceChart/TurnoverChart
5. 图表渲染

Tab 切换流程：
1. 用户点击 Tab
2. handleTabChange 更新 activeTab
3. 保存到 localStorage
4. 条件渲染对应图表组件
5. 图表初始化/销毁
```

### 1.5 潜在风险点

| 风险点 | 风险等级 | 影响 | 缓解措施 |
|--------|---------|------|---------|
| 图表初始化时机不当 | 高 | 刷新后图表显示异常 | 使用 requestAnimationFrame |
| 内存泄漏 | 高 | 长时间运行后性能下降 | 组件卸载时销毁图表实例 |
| localStorage 缓存失效 | 中 | 数据不一致 | 24 小时缓存过期机制 |
| Tab 切换时图表未正确销毁 | 中 | 内存泄漏、显示异常 | 完善的 cleanup 逻辑 |
| 快速刷新导致状态混乱 | 中 | UI 状态不一致 | 防抖处理、状态锁 |
| 浏览器兼容性问题 | 中 | 部分浏览器功能异常 | 主流浏览器测试 |
