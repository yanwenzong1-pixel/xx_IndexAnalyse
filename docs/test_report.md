## 三、问题清单

### 3.1 高优先级问题（必须修复）

#### BUG-001: 缓存 TTL 配置与需求不符

| 项目 | 内容 |
|------|------|
| **问题描述** | CacheManager 未实现按缓存类型区分 TTL，所有缓存使用相同的 TTL 配置 |
| **需求规格** | 影响系数 5 分钟、风险评级 30 分钟、风险原因 24 小时、基础数据 1 分钟 |
| **代码现状** | cacheConfig 定义了 4 种 TTL，但 isExpired() 方法未使用 type 参数，统一使用 key 作为 TTL 查找 |
| **复现步骤** | 1. 打开控制台<br>2. 执行：`cacheManager.set('test_impact', {data: 1}, 'impactCoefficient')`<br>3. 执行：`cacheManager.set('test_reason', {data: 2}, 'riskReason')`<br>4. 5 分钟后检查两个缓存的过期状态 |
| **预期结果** | test_impact 过期，test_reason 未过期 |
| **实际结果** | 两个缓存同时过期或同时不过期 |
| **影响范围** | 所有缓存相关的功能 |
| **修复建议** | 修改 isExpired() 方法，根据缓存 key 的类型使用不同的 TTL |
| **严重程度** | 🔴 高 |

**代码定位**: `index.html:234-241`

```javascript
// 当前代码
isExpired(key) {
  const cached = this.get(key);
  if (!cached) return true;
  
  const ttl = this.cacheConfig[key] || 60000;  // ❌ 问题：使用 key 查找 TTL，应该根据类型
  const now = Date.now();
  return (now - cached.timestamp) > ttl;
}

// 修复建议
isExpired(key) {
  const cached = this.get(key);
  if (!cached) return true;
  
  // 根据 key 前缀判断类型
  let type = 'baseData';
  if (key.startsWith('impact_')) type = 'impactCoefficient';
  else if (key.startsWith('risk_level_')) type = 'riskLevel';
  else if (key.startsWith('risk_reason_')) type = 'riskReason';
  
  const ttl = this.cacheConfig[type] || 60000;
  const now = Date.now();
  return (now - cached.timestamp) > ttl;
}
```

**代码定位**: `index.html:358-365`

```javascript
// 当前代码
const getColor = (val) => {
  if (val < -0.5) return '#E53E3E';      // 深红
  if (val < -0.2) return '#F3674C';      // 橙红
  if (val < 0.2) return '#718096';       // 灰色
  if (val < 0.5) return '#68D391';       // 浅绿

// 修复建议（假设影响系数范围是 0-100）
const getColor = (val) => {
```
