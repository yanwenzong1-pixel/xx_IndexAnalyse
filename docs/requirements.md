# 维度深度分析模块 - 需求规格说明书

## 1. 需求概述

### 1.1 项目背景
在现有"维度深度分析"模块基础上，增加影响系数、风险评级和风险原因三个数据项的展示，提升用户对各维度风险的直观理解。

### 1.2 开发目标
改造 DimensionAnalysis 组件，为五大维度（资金结构、流动性、估值与业绩、政策与制度、宏观环境）的每个指标增加三个新字段：
- **影响系数**：量化指标对整体风险的影响程度（0-100%）
- **风险评级**：直观的风险等级标识（低/中/高）
- **风险原因**：简要说明风险来源

## 2. 功能需求

### 2.1 数据展示需求

#### 2.1.1 影响系数
- **显示格式**：百分比（保留 1 位小数）
- **颜色映射**：
  - 0-30%：绿色（低风险）
  - 31-70%：黄色（中风险）
  - 71-100%：红色（高风险）
- **展示形式**：环形进度条（SVG 实现）

#### 2.1.2 风险评级
- **评级等级**：低风险、中风险、高风险
- **颜色映射**：
  - 低风险：绿色 (#10b981)
  - 中风险：黄色 (#f59e0b)
  - 高风险：红色 (#ef4444)
- **展示形式**：徽章 Badge + 文字说明

#### 2.1.3 风险原因
- **显示格式**：简短文本（最多 50 字）
- **交互**：默认显示前 20 字，点击展开/收起
- **动画**：平滑过渡效果（200ms）

### 2.2 状态管理需求

#### 2.2.1 数据结构
每个维度指标包含以下字段：
```javascript
{
  // 原有字段
  value: number,        // 指标值
  unit: string,         // 单位
  description: string,  // 解读说明
  
  // 新增字段
  impactCoefficient: number,  // 影响系数 (0-100)
  riskLevel: 'low' | 'medium' | 'high',  // 风险评级
  riskReason: string,    // 风险原因
  lastUpdated: number    // 最后更新时间戳
}
```

#### 2.2.2 缓存机制
| 数据项 | 缓存时间 | 缓存键前缀 | 刷新策略 |
|--------|---------|-----------|---------|
| 影响系数 | 5 分钟 | `impact_` | 超时自动刷新 |
| 风险评级 | 30 分钟 | `risk_level_` | 超时自动刷新 |
| 风险原因 | 24 小时 | `risk_reason_` | 超时自动刷新 |
| 基础数据 | 1 分钟 | `dimension_` | 手动刷新 |

#### 2.2.3 状态保持
- 页面刷新后，从 localStorage 恢复缓存数据
- 显示"缓存数据"标识（如适用）
- 自动检查缓存过期时间

### 2.3 交互需求

#### 2.3.1 展开/收起
- 风险原因文本过长时显示"展开"按钮
- 点击后平滑展开显示完整内容
- 再次点击收起

#### 2.3.2 数据刷新
- 继承现有刷新按钮功能
- 刷新时显示加载状态
- 刷新成功后更新缓存

#### 2.3.3 Tab 切换
- 切换到其他 Tab 时保存当前状态
- 切换回来时恢复状态
- 确保数据一致性

## 3. 非功能需求

### 3.1 性能要求
- 页面加载时间 < 2 秒
- 缓存读取时间 < 100ms
- 展开/收起动画帧率 > 50fps

### 3.2 兼容性要求
- Chrome 90+
- Firefox 88+
- Safari 14+
- Edge 90+

### 3.3 安全要求
- 所有用户输入进行 XSS 过滤
- localStorage 数据加密存储（可选）

## 4. 验收标准

### 4.1 功能验收
- [ ] 五大维度卡片样式统一
- [ ] 影响系数环形进度条正确显示
- [ ] 风险评级颜色映射正确
- [ ] 风险原因展开/收起功能正常
- [ ] 缓存机制按指定时间生效

### 4.2 状态管理验收
- [ ] 页面刷新后数据状态保持
- [ ] Tab 切换时状态一致
- [ ] 缓存过期自动刷新

### 4.3 响应式验收
- [ ] 桌面端（>1024px）：5 列布局
- [ ] 平板端（640-1024px）：2-3 列布局
- [ ] 移动端（<640px）：1 列布局

## 5. 技术约束

### 5.1 开发约束
- 单 HTML 文件架构（React 18 + Babel）
- 使用 Tailwind CSS 进行样式设计
- 遵循现有代码规范

### 5.2 依赖约束
- 不引入新的外部库
- SVG 环形进度条手动实现
- 使用原生 localStorage API

## 6. 风险评估

### 6.1 技术风险
- **缓存管理复杂性**：多缓存周期可能导致状态不一致
  - 缓解：统一缓存管理工具函数
- **状态保持**：页面刷新后状态恢复可能失败
  - 缓解：完善的错误处理和降级策略

### 6.2 进度风险
- **开发工作量**：预计 2-3 天
- **测试工作量**：预计 1 天

## 7. 附录

### 7.1 术语表
- **影响系数**：指标对整体风险的影响权重
- **风险评级**：基于影响系数和其他因素的综合评估
- **风险原因**：导致风险评级的主要因素说明
