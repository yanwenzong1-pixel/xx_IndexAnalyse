### 1.4 缓存机制（40/100 - ❌ 严重不符）

#### 需求规格（PRD 2.2.2）
| 数据项 | 缓存时间 | 缓存键前缀 |
|--------|---------|-----------|
| 影响系数 | **5 分钟** | `impact_` |
| 风险评级 | **30 分钟** | `risk_level_` |
| 风险原因 | **24 小时** | `risk_reason_` |
| 基础数据 | **1 分钟** | `dimension_` |

#### 实际实现
```javascript
// index.html:198-203
this.cacheConfig = {
  impactCoefficient: 5 * 60 * 1000,     // 5 分钟 ✅
  riskLevel: 30 * 60 * 1000,            // 30 分钟 ✅
  riskReason: 24 * 60 * 60 * 1000,      // 24 小时 ✅
  baseData: 1 * 60 * 1000               // 1 分钟 ✅
};

// index.html:234-241 (isExpired 方法)
isExpired(key) {
  const cached = this.get(key);
  if (!cached) return true;
  
  const ttl = this.cacheConfig[key] || 60000;  // ❌ 问题：使用 key 查找 TTL
  const now = Date.now();
  return (now - cached.timestamp) > ttl;
}
```

#### 问题清单
| 编号 | 问题描述 | 严重程度 | 违反需求 |
|------|----------|----------|----------|
| REQ-010 | CacheManager 未实现按缓存类型区分 TTL | 🔴 致命 | PRD 2.2.2 |
| REQ-011 | 所有缓存使用相同 TTL（60 秒默认值） | 🔴 致命 | PRD 2.2.2 |
| REQ-012 | 缓存键未使用前缀（如 `impact_`） | 🟠 严重 | PRD 2.2.2 |

#### 问题分析
**核心缺陷**: `isExpired()` 方法使用 `key` 直接查找 TTL，但实际设置缓存时传入的 key 是自定义键（如 `'test_impact'`），而非缓存类型键（如 `'impactCoefficient'`）。

**复现步骤**:
1. 打开浏览器控制台
2. 执行：`cacheManager.set('my_impact_data', {value: 1}, 'impactCoefficient')`
3. 执行：`cacheManager.set('my_reason_data', {value: 2}, 'riskReason')`
4. 等待 5 分钟后检查
5. **预期**: `my_impact_data` 过期，`my_reason_data` 未过期
6. **实际**: 两个缓存同时过期（都使用 60 秒默认 TTL）

#### 整改要求
**必须修改**:
1. 修改 `isExpired()` 方法，根据缓存 key 的类型使用不同的 TTL
2. 修改 `set()` 方法，增加 `type` 参数用于指定缓存类型
3. 或修改缓存键命名规范，使用前缀标识类型（如 `impact_xxx`）

**修复代码示例**:
```javascript
// 方案 1：增加 type 参数
set(key, data, type) {
  try {
    const cacheData = {
      data: data,
      timestamp: Date.now(),
      type: type  // 保存类型
    };
    localStorage.setItem(key, JSON.stringify(cacheData));
  } catch (e) {
    console.warn('缓存设置失败:', e);
  }
}

isExpired(key) {
  const cached = this.get(key);
  if (!cached) return true;
  
  // 根据缓存类型获取 TTL
  const ttl = this.cacheConfig[cached.type] || 60000;
  const now = Date.now();
  return (now - cached.timestamp) > ttl;
}

// 方案 2：根据 key 前缀判断类型
isExpired(key) {
  const cached = this.get(key);
  if (!cached) return true;
  
  let type = 'baseData';
  if (key.startsWith('impact_')) type = 'impactCoefficient';
  else if (key.startsWith('risk_level_')) type = 'riskLevel';
  else if (key.startsWith('risk_reason_')) type = 'riskReason';
  
  const ttl = this.cacheConfig[type] || 60000;
  const now = Date.now();
  return (now - cached.timestamp) > ttl;
}
```

**代码位置**: `index.html:207-241`
