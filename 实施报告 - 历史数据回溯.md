### 2. 前端开发（4 个任务）

#### T007：API 调用函数 ✅
**文件**：`web/frontend/index.html`

**核心函数**：
```javascript
const fetchPredictionData = async () => {
  try {
    const response = await fetch('http://localhost:5000/api/predict/history?days=100');
    
    if (!response.ok) {
      throw new Error(`HTTP error! status: ${response.status}`);
    }
    
    const result = await response.json();
    
    if (result.success && result.data && result.data.length > 0) {
      // 转换后端数据格式为前端所需格式
      const formattedData = result.data.map(item => ({
        date: item.date,
        t1Prob: parseFloat((item.prob_t1 * 100).toFixed(2)),
        t5Prob: parseFloat((item.prob_t5 * 100).toFixed(2))
      }));
      return formattedData;
    }
  } catch (err) {
    console.error('API 调用失败，使用降级数据:', err);
    return null;
  }
};
```

if (cachedData && cachedTimestamp) {

## 技术亮点

### 1. 避免未来函数
**问题**：因子计算需要 252 天历史窗口，必须确保不使用未来数据

**解决方案**：
```python
for i in range(actual_days):
    current_idx = end_idx - i
    # 截取到当前日期的历史数据（避免未来函数）
    df_up_to_current = extended_df.iloc[:current_idx + 1].copy()
    # 计算单日预测
    daily_result = calculate_single_day_prediction(df_up_to_current)
```

### 2. 分层缓存策略
**三级缓存**：
1. **内存缓存**（可选）：Redis 缓存每日计算结果
2. **浏览器缓存**：localStorage 缓存 24 小时
3. **降级方案**：模拟数据作为最后备选

**优势**：
- 首次加载：API 调用（~300-500ms）
- 后续访问：缓存命中（< 100ms）
- API 失败：降级可用

### 3. 优雅降级
**降级流程**：
```
API 调用成功 → 使用真实数据 ✅
     ↓ 失败
缓存有效 → 使用缓存数据 ⚠️
     ↓ 失效
生成模拟数据 → 显示错误提示 ❌
```
