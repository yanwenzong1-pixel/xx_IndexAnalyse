# 技术实现文档 - 价格指数图表刷新修复

## 1. 问题描述

### 1.1 问题现象
- F5 刷新页面后，价格指数图表有概率绘制错误
- 图表可能显示空白、数据不完整或样式异常
- 问题出现概率约为 20-30%

### 1.2 根本原因分析

#### 1.2.1 useEffect 依赖检查不严格
**问题代码**：
```javascript
useEffect(() => {
  // ... 图表初始化逻辑
}, [data]);
```

**问题分析**：
- React 使用浅层比较（Object.is）检测依赖变化
- 数组是引用类型，即使内容相同，引用不同也会触发
- 反之，某些情况下引用相同但内容变化可能不触发

#### 1.2.2 图表实例状态管理不当
**问题代码**：
```javascript
const chartInstance = useRef(null);

if (!chartInstance.current) {
  chartInstance.current = echarts.init(chartRef.current);
}
```

**问题分析**：
- `chartInstance.current` 在组件重新渲染时保持旧引用
- DOM 元素可能被 React 回收重建，但 ref 仍指向旧实例
- 导致在无效 DOM 上操作图表

#### 1.2.3 数据验证逻辑可能误判
**问题代码**：
```javascript
const validData = data.filter(item => 
  item && typeof item === 'object' && item.date && typeof item.close === 'number'
);

if (validData.length === 0) {
  throw new Error('无效的图表数据');
}
```

**问题分析**：
- 验证逻辑过于简单，未检查 NaN 和 Infinity
- 未考虑数据顺序变化
- 错误处理机制不完善

#### 1.2.4 缺少强制刷新机制
**问题分析**：
- 页面刷新后无法主动触发表格重绘
- 依赖数据变化被动更新
- 无法处理 DOM 状态异常

## 2. 解决方案

### 2.1 优化 useEffect 依赖管理

#### 2.1.1 使用 useCallback 包装函数
```javascript
const validateData = useCallback((data) => {
  // 验证逻辑
}, []);

const calculateMA = useCallback((data, period) => {
  // 计算逻辑
}, []);

const renderChart = useCallback(() => {
  // 渲染逻辑
}, [data, validateData, calculateMA]);
```

**优势**：
- 函数引用稳定，避免不必要的重渲染
- 依赖关系明确，易于维护

#### 2.1.2 分离关注点的多个 useEffect
```javascript
// 主渲染逻辑
useEffect(() => {
  const rafId = requestAnimationFrame(() => {
    const success = renderChart();
    if (!success) {
      setTimeout(renderChart, 200);
    }
  });
  
  const resizeHandler = () => {
    chartInstance.current?.resize();
  };
  
  window.addEventListener('resize', resizeHandler);
  
  return () => {
    cancelAnimationFrame(rafId);
    window.removeEventListener('resize', resizeHandler);
  };
}, [renderChart]);

// 强制刷新监听
useEffect(() => {
  if (forceRefreshKey !== undefined) {
    forceRefresh();
  }
}, [forceRefreshKey, forceRefresh]);

// 组件卸载清理
useEffect(() => {
  return () => {
    if (chartInstance.current) {
      chartInstance.current.dispose();
      chartInstance.current = null;
    }
  };
}, []);
```

### 2.3 添加数据深度比较逻辑

#### 2.3.1 实现深度比较函数
```javascript
const deepCompareArrays = (arr1, arr2) => {
  if (!arr1 || !arr2 || !Array.isArray(arr1) || !Array.isArray(arr2)) {
    return false;
  }
  
  if (arr1.length !== arr2.length) {
    return false;
  }
  
  for (let i = 0; i < arr1.length; i++) {
    const item1 = arr1[i];
    const item2 = arr2[i];
    
    if (typeof item1 !== typeof item2) {
      return false;
    }
    
    if (item1 && typeof item1 === 'object' && item2 && typeof item2 === 'object') {
      const keys1 = Object.keys(item1).sort();
      const keys2 = Object.keys(item2).sort();
      
      if (keys1.length !== keys2.length) {
        return false;
      }
      
      for (let j = 0; j < keys1.length; j++) {
        if (keys1[j] !== keys2[j] || item1[keys1[j]] !== item2[keys2[j]]) {
          return false;
        }
      }
    } else if (item1 !== item2) {
      return false;
    }
  }
  
  return true;
};
```

#### 2.3.2 增强的数据验证
```javascript
const validateData = useCallback((data) => {
  if (!data || !Array.isArray(data) || data.length === 0) {
    return { isValid: false, validData: [], error: '数据为空或格式错误' };
  }

  const validData = data.filter(item => 
    item && 
    typeof item === 'object' && 
    item.date && 
    typeof item.close === 'number' &&
    !isNaN(item.close) &&
    isFinite(item.close)
  );

  if (validData.length === 0) {
    return { isValid: false, validData: [], error: '无有效的图表数据' };
  }

  // 检查数据是否有变化
  if (previousData.current && deepCompareArrays(previousData.current, data)) {
    return { isValid: false, validData: [], error: '数据无变化', noChange: true };
  }

  return { isValid: true, validData, error: null };
}, []);
```

### 2.4 实现强制刷新机制

#### 2.4.1 父组件传递刷新 key
```javascript
// App.jsx
const [refreshKey, setRefreshKey] = useState(0);

const handleRefresh = async () => {
  try {
    // ... 刷新数据
    
    // 增加刷新 key，强制图表重绘
    setRefreshKey(prev => prev + 1);
  } catch (err) {
    // ... 错误处理
  }
};

// 页面加载后强制刷新一次
useEffect(() => {
  const timer = setTimeout(() => {
    setRefreshKey(prev => prev + 1);
  }, 100);
  
  return () => clearTimeout(timer);
}, []);

// 使用
<PriceChart key={refreshKey} data={data.history} forceRefreshKey={refreshKey} />
```

#### 2.4.2 子组件监听刷新信号
```javascript
const PriceChart = ({ data, forceRefreshKey }) => {
  // ... 其他逻辑
  
  useEffect(() => {
    if (forceRefreshKey !== undefined) {
      console.log('检测到强制刷新信号，key:', forceRefreshKey);
      forceRefresh();
    }
  }, [forceRefreshKey, forceRefresh]);
  
  // ...
};
```

## 3. 技术架构

### 3.1 组件结构
```
App.jsx
├── refreshKey (state)
├── handleRefresh (刷新处理器)
└── PriceChart
    ├── data (props)
    ├── forceRefreshKey (props)
    ├── chartRef (ref)
    ├── chartInstance (ref)
    ├── previousData (ref)
    ├── validateData (useCallback)
    ├── calculateMA (useCallback)
    ├── renderChart (useCallback)
    └── forceRefresh (useCallback)
```

### 3.2 数据流
```
用户操作/F5 刷新
    ↓
App.handleRefresh
    ↓
setRefreshKey(prev => prev + 1)
    ↓
PriceChart.forceRefreshKey 变化
    ↓
forceRefresh() 执行
    ↓
销毁旧实例 → 创建新实例 → 渲染图表
```

### 3.3 生命周期管理
```
组件挂载
    ↓
useEffect(setup)
    ↓
requestAnimationFrame → renderChart()
    ↓
[数据变化/强制刷新]
    ↓
renderChart() 重新执行
    ↓
组件卸载
    ↓
useEffect(cleanup) → dispose()
```
