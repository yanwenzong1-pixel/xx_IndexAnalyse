# 微盘股指数分析系统PRD文档

## 1. 产品概览
微盘股指数分析系统是一个实时监控和分析微盘股指数的前端应用，提供指数价格走势、换手率、成交额等关键指标的可视化展示，帮助用户快速了解市场动态。

## 2. 核心功能

### 2.1 功能模块

| 模块名称 | 功能描述 |
|---------|----------|
| 顶部导航栏 | 显示系统标题、风险等级和刷新按钮 |
| 预警信息 | 显示风险预警信息（当风险等级达到预警阈值时） |
| 指标卡片 | 展示收盘价、涨跌幅、成交额、换手率等关键指标 |
| 价格走势图表 | 展示微盘股指数的历史价格走势，包含MA5和MA20均线 |
| 换手率和成交额图表 | 展示历史换手率和成交额数据 |

### 2.2 页面刷新状态管理

#### 2.2.1 功能描述
- **页面刷新**：用户可以通过点击刷新按钮手动刷新数据
- **自动刷新**：系统每3小时自动刷新数据
- **加载状态**：在数据加载过程中显示加载状态
- **错误处理**：当数据加载失败时显示错误信息并提供重试机制

#### 2.2.2 交互规则
- 页面初始加载时显示加载状态，直至数据加载完成
- 点击刷新按钮时，按钮显示"刷新中..."并禁用，直至刷新完成
- 数据加载失败时，显示错误信息并提供重试按钮
- 数据加载过程中，指标卡片和图表区域显示加载中的骨架屏
- 页面刷新后，所有数据和图表应正确显示，无异常状态

### 2.3 图表渲染优化

#### 2.3.1 功能描述
- **图表初始化**：确保在 DOM 渲染完成后初始化图表
- **容器尺寸检测**：初始化前验证容器尺寸（宽度和高度）是否大于 0
- **数据依赖**：图表渲染依赖于数据加载完成
- **响应式**：图表能够响应窗口大小变化
- **内存管理**：在组件卸载时销毁图表实例，避免内存泄漏

#### 2.3.2 交互规则
- 页面刷新后，图表应正确渲染，无空白或异常显示
- Tab 切换后，图表应保持正常显示状态
- 窗口大小变化时，图表应自动调整大小
- 图表容器必须具有明确的尺寸（非 0）才能初始化

### 2.4 图表容器尺寸问题修复

#### 2.4.1 问题描述
**问题现象**：
- 价格指数图表在某些场景下显示为一条水平线
- 需要缩放页面或切换 Tab 才能正确显示
- 成交量图表也可能出现类似问题

**问题根因**：
- 图表容器在 ECharts 初始化时高度为 0 或未完全渲染
- ECharts 根据错误的容器尺寸（0 高度）初始化，导致图表显示异常
- Tab 切换或窗口缩放触发重新渲染，容器尺寸正确后图表恢复正常

**影响范围**：
- 页面初始加载时
- Tab 切换时
- 页面刷新时
- 父容器尺寸动态变化时

#### 2.4.2 解决方案
- **容器尺寸检测**：初始化前检查容器 offsetWidth 和 offsetHeight 是否大于 0
- **延迟重试机制**：容器尺寸为 0 时，延迟重试（最多重试 10 次，每次间隔 100ms）
- **Tab 切换监听**：Tab 切换时强制重新检测容器尺寸并重新渲染
- **ResizeObserver 监听**：使用 ResizeObserver 监听容器尺寸变化，自动触发重绘

#### 2.4.3 交互规则
- 容器尺寸为 0 时，不初始化图表，等待容器尺寸正确
- 容器尺寸恢复后，自动初始化图表
- 图表初始化失败时，在控制台输出错误日志，但不影响页面其他功能

## 3. 核心流程

### 3.1 页面加载流程
1. 用户打开页面
2. 系统显示加载状态
3. 系统请求数据
4. 数据加载完成后，系统渲染页面内容
5. 初始化图表组件
6. 显示完整页面

### 3.2 数据刷新流程
1. 用户点击刷新按钮
2. 系统显示加载状态，禁用刷新按钮
3. 系统请求新数据
4. 数据加载完成后，更新页面内容
5. 重新渲染图表
6. 恢复刷新按钮状态

## 4. 页面设计

### 4.1 布局结构
- 顶部导航栏：固定在页面顶部，包含系统标题、风险等级和刷新按钮
- 预警信息：位于导航栏下方，仅当有预警时显示
- 指标卡片：位于预警信息下方，以网格布局展示四个关键指标
- 图表区域：位于指标卡片下方，包含价格走势图表和换手率成交额图表
- 页脚：固定在页面底部，显示版权信息

### 4.2 响应式设计
- 桌面端：指标卡片以4列网格布局显示，图表以1列布局显示
- 平板端：指标卡片以2列网格布局显示，图表以1列布局显示
- 移动端：指标卡片以1列网格布局显示，图表以1列布局显示

## 6. 风险评估

### 6.1 技术风险
- **图表初始化时机**：如果图表初始化时机不当，可能导致页面刷新后图表显示异常
- **数据加载失败**：如果数据加载失败，可能导致页面显示异常
- **内存泄漏**：如果图表实例未正确销毁，可能导致内存泄漏

### 6.2 解决方案
- 使用requestAnimationFrame确保DOM渲染完成后初始化图表
- 添加完整的错误处理和数据结构检查
- 在组件卸载时销毁图表实例，避免内存泄漏
- 添加加载状态显示，提升用户体验
