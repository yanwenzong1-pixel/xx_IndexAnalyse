# 数据初始化问题诊断指南

## 问题描述
```
[RiskTab] API 返回错误: {error: 'data_not_initialized', message: '基础数据未初始化', success: false}
```

## 可能的原因

### 1. 后端服务未完全启动
数据初始化需要时间，可能在初始化完成前就收到了请求。

### 2. 数据获取失败
网络问题或数据源问题导致无法获取微盘股指数数据。

### 3. 初始化异常
代码执行过程中出现未捕获的异常。

## 诊断步骤

### 步骤 1：手动启动后端并查看日志

打开新的终端窗口，执行：

```bash
cd C:\Users\User\Documents\xx_IndexAnalyse
```

如果看到 `⚠️ 数据初始化失败`，说明数据获取有问题。

运行测试脚本：

```bash
python test_data_init.py
```

**预期输出**：
```

获取数据...
✅ 数据获取成功！
```

### 步骤 3：检查网络连接

数据获取需要访问东方财富网 API。

**测试命令**：
```bash
ping push2his.eastmoney.com
```

如果无法访问，可能是网络问题。

### 步骤 4：手动测试 API

启动后端服务后，在浏览器访问：

```
http://localhost:5000/api/health
```

**预期响应**：
```json
{
  "success": true,
  "message": "服务运行正常",
  "version": "2.1.0"
}
```

如果返回错误或无法访问，说明后端有问题。

## 解决方案

### 方案 1：等待初始化完成

后端服务启动后，等待 10-20 秒再刷新前端页面。

### 方案 2：重启后端服务

1. 停止当前运行的后端服务（Ctrl+C）
2. 重新启动：
   ```bash
   python web/backend/simple_app.py
   ```
3. 等待看到"✅ 数据初始化成功"
4. 刷新前端页面

### 方案 3：检查防火墙

确保防火墙没有阻止 Python 访问网络。

### 方案 4：检查数据源

如果数据源 API 变更，可能需要更新代码。

当前使用的数据源：
- 东方财富网 API
- 证券代码：90.BK1158（微盘股指数）

## 快速诊断脚本

创建文件 `quick_diagnose.py`：

```python
# -*- coding: utf-8 -*-
import requests
import time

print("诊断中...")

# 1. 检查后端服务
try:
    response = requests.get('http://localhost:5000/api/health', timeout=5)
    print("✅ 后端服务可访问")
    print(f"   响应：{response.json()}")
except:
    print("❌ 后端服务不可访问")
    print("   请先启动后端：python web/backend/simple_app.py")
    exit(1)

# 2. 等待初始化
print("\n等待数据初始化...")
time.sleep(5)

# 3. 测试风险接口
try:
    response = requests.get('http://localhost:5000/api/risk/current', timeout=10)
    result = response.json()
    if result.get('success'):
        print("✅ 风险服务正常")
        print(f"   日期：{result['data']['date']}")
        print(f"   风险分：{result['data']['risk_score']}")
    else:
        print(f"❌ 风险服务异常：{result}")
except Exception as e:
    print(f"❌ 风险服务测试失败：{e}")

print("\n诊断完成！")
```

运行：
```bash
python quick_diagnose.py
```

## 常见问题

### Q1: 数据获取超时
**A**: 网络问题，稍后重试或检查网络连接。

### Q2: 端口 5000 被占用
**A**: 
```bash
# 查找并终止占用进程
netstat -ano | findstr :5000
taskkill /PID <进程 ID> /F
```

### Q3: 模块导入错误
**A**: 确保所有导入错误已修复（见导入错误修复指南）。

### Q4: CORS 错误
**A**: 确保已添加 CORS 支持（见 CORS 修复指南）。
