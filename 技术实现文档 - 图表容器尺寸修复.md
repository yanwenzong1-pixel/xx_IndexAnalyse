# 技术实现文档 - 图表容器尺寸修复

## 问题回顾

### 原始问题
微盘股指数价格走势图在 F5 刷新后有概率显示为一条水平线，但切换 Tab 或缩放页面后图表恢复正常。

### 关键线索
- **切换 Tab** → 图表正常 ✅
- **缩放页面** → 图表正常 ✅
- **数据是正确的**（tooltip 显示正确数值）

### 问题根因

**图表容器在初始化时高度为 0 或未完全渲染**

```javascript
// 第 699 行
<div ref={chartRef} className="chart-container" style={{ width: '100%', height: '100%' }}></div>
```

**问题分析**：
1. `PriceChart` 容器的 `height: 100%` 是相对于父元素的
2. 父元素虽然有 `h-[400px]`，但在组件挂载瞬间，可能还未渲染完成
3. ECharts 初始化时，如果容器高度为 0，图表会被压缩成一条线
4. **切换 Tab** → 组件重新挂载 → 容器已渲染完成 → 高度正确 → 图表正常
5. **缩放页面** → 触发 `window.resize` → 调用 `chartInstance.resize()` → 图表重新计算尺寸 → 图表正常

---

## 解决方案

### 1. 容器尺寸检测

```javascript
// 检测容器尺寸是否准备好
const checkContainerSize = () => {
  const rect = chartRef.current.getBoundingClientRect();
  return rect.width > 0 && rect.height > 0;
};
```

**原理**：
- 使用 `getBoundingClientRect()` 获取容器实际尺寸
- 检查宽度和高度是否都 > 0
- 确保容器已完全渲染

### 2. 延迟重试机制

```javascript
// 等待容器尺寸准备好
const waitForContainer = (callback, retries = 10, delay = 100) => {
  if (checkContainerSize()) {
    callback();
    return;
  }
  
  if (retries <= 0) {
    console.warn('容器尺寸检测超时，强制初始化');
    callback();
    return;
  }
  
  setTimeout(() => {
    waitForContainer(callback, retries - 1, delay);
  }, delay);
};
```

**原理**：
- 递归检查容器尺寸，最多重试 10 次
- 每次延迟 100ms，总等待时间最多 1 秒
- 超时后强制初始化（兜底方案）

### 3. ResizeObserver 监听

```javascript
// 使用 ResizeObserver 监听容器尺寸变化
const resizeObserver = new ResizeObserver(() => {
  if (chartInstance.current) {
    chartInstance.current.resize();
  }
});

if (chartRef.current) {
  resizeObserver.observe(chartRef.current);
}
```

**原理**：
- `ResizeObserver` 是现代浏览器 API，监听元素尺寸变化
- 当容器尺寸变化时，自动调用 `chartInstance.resize()`
- 确保图表始终与容器尺寸同步

### 4. 完整的初始化流程

```javascript
React.useEffect(() => {
  if (!chartRef.current || !data || data.length === 0) {
    return;
  }

  // 等待容器准备好后初始化
  waitForContainer(initChart);
}, [data]);
```

**流程**：
1. 组件挂载
2. useEffect 执行
3. 检查容器尺寸
4. 如果未准备好，延迟重试
5. 容器准备好后，初始化图表
6. 使用 ResizeObserver 监听尺寸变化
