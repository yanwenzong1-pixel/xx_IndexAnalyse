## 二、根本原因分析

### 2.1 核心原因

**两个数值使用了完全不同的数据源和计算逻辑：**

| 特征 | 最新下跌概率（51%） | 近 100 日趋势图最新值（~80%） |
|------|-------------------|---------------------------|
| **数据源** | 后端 API 真实计算 | 前端模拟数据（降级方案） |
| **计算方式** | 25 因子 Logit 回归模型 | 固定算法随机数生成 |
| **API 端点** | `/api/predict` | `/api/predict/history`（降级为模拟） |
| **位置** | 综合风险评估模块 | 下跌风险预测模块 |
| **真实性** | ✅ 真实市场数据 | ⚠️ 模拟示意图 |

### 2.2 详细技术原因

#### （1）最新下跌概率（51%）的计算流程

```
用户请求 → 前端调用 /api/predict → 后端计算：
  1. 获取 25 因子数据（流动性、资金结构、估值等）
  2. 计算综合风险分（base_risk, adjusted_risk）
  3. 应用 Logit 回归模型：
     ln(P/(1-P)) = α₀ + α₁·R_adjusted + α₂·R_adjusted² 
                  + α₃·Volatility_20d + α₄·Momentum_1d
  4. 返回真实概率值：42%（次日）/ 58%（5 日）
```

**代码位置**：
- 后端：`c:\Users\User\Documents\xx_IndexAnalyse\web\backend\utils\risk_calculation_service.py` 第 319-380 行
- 前端调用：`c:\Users\User\Documents\xx_IndexAnalyse\web\frontend\index.html` 第 3462-3478 行

#### （2）近 100 日趋势图最新值（~80%）的计算流程

```
用户请求 → 前端尝试调用 /api/predict/history?days=100
  ↓
API 调用失败（服务未启动或其他原因）
  ↓
降级到前端模拟数据生成：
  const dateSeed = parseInt(dateStr.replace(/-/g, '')); // YYYYMMDD
  const t1Prob = seededRandom(dateSeed) * 60 + 20; // 20-80%
  
对于今日（2026-03-13）：
  dateSeed = 20260313
  seededRandom(20260313) ≈ 0.996
  t1Prob = 0.996 * 60 + 20 ≈ 79.8% ≈ 80%
```

**代码位置**：
- 前端模拟数据生成：`c:\Users\User\Documents\xx_IndexAnalyse\web\frontend\index.html` 第 1337-1367 行
- 降级逻辑：第 1422-1434 行

### 2.3 为什么 API 调用会失败

根据代码逻辑（第 1370-1396 行），API 调用失败可能有以下原因：

1. **后端服务未启动**：`http://localhost:5000` 无法访问
2. **历史回溯接口未实现完整**：`/api/predict/history` 返回错误
3. **网络超时或 CORS 问题**：跨域请求被阻止
4. **数据初始化失败**：`history_service` 为 `null`

当 API 调用失败时，前端会自动降级到模拟数据（第 1431-1433 行）：

```javascript
} else {
  // API 失败，使用降级模拟数据
  newData = generateFallbackData();
  console.log('⚠️ 使用降级模拟数据');
  setError('实时数据获取失败，当前显示为模拟数据');
}
```
